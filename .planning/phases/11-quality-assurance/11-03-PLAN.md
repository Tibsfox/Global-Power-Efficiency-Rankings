Extract all numeric labor market claims from index.html for verification:

1. Extract salary ranges:
```bash
grep -n -oP '\$[0-9]{2,3}K-[0-9]{2,3}K[^<]*' index.html > .planning/phases/11-quality-assurance/salary-ranges.txt
```

2. Extract percentage claims (job growth, market growth):
```bash
grep -n -oP '[0-9]{1,3}(-[0-9]{1,3})?%\s*[a-zA-Z ]{3,30}' index.html > .planning/phases/11-quality-assurance/percentage-claims.txt
```

3. Extract market size figures:
```bash
grep -n -oP '\$?[0-9]+(\.[0-9]+)?\s*(million|billion|trillion)' index.html > .planning/phases/11-quality-assurance/market-sizes.txt
```

4. Create claims audit report:

   - Note whether claim has existing caveat

<task type="auto">
  <name>Task 2: Add caveats and moderate claims</name>
  <files>index.html</files>
  <action>
For each claim identified in claims-audit.txt as needing action:

1. **Add inline caveats to salary ranges** that lack them:
   - Pattern: `$XXK-XXXK` → `$XXK-XXXK (U.S., varies by location and experience)`
   - Only add if caveat doesn't already exist
   - Keep existing caveats if present

2. **Add context to percentage claims**:
   - Job growth: "X% projected growth (BLS, varies by region)"
   - Market growth: "X% annual growth (industry estimates)"

3. **Moderate inflated claims**:
   - If salary range appears too high for role, adjust to BLS-aligned range
   - Document adjustment in commit message
   - Example: "$180K-250K" for entry-level → "$85K-120K (varies by location and experience)"

4. **Add section-level disclaimer** if not present:
   - Check if general disclaimer exists near salary information
   - If missing, add: "Salary ranges represent U.S. averages and vary significantly by location, experience level, organization type, and market conditions."

5. Batch approach (efficient):
   - For simple caveat additions, use sed for bulk update:
   ```bash
   # Add caveat to salary ranges that don't have one (careful with existing caveats)
   # This is a pattern example - actual implementation should preserve existing caveats
   ```
   - For inflated claims, edit manually to preserve context

6. After all changes, spot-check 10-15 claims to verify:
   - Caveats added correctly
   - No double-caveats
   - Formatting preserved

7. Commit the changes:
```bash
git add index.html
git commit -m "fix(qa): add caveats to labor market claims

Added inline caveats to [N] salary ranges: '(varies by location/experience)'
Moderated [N] inflated claims to realistic ranges:
- [Example adjustments if any]

Claims verified against BLS.gov Occupational Employment Statistics.

Co-Authored-By: Claude Opus 4.5 <noreply@anthropic.com>"
```

<task type="auto">
  <name>Task 1: Extract and validate all URLs</name>
  <files>index.html, .planning/phases/11-quality-assurance/urls.txt, .planning/phases/11-quality-assurance/url-validation-results.txt</files>
  <action>
Extract all unique URLs from index.html and validate them using parallel HTTP requests:

1. Extract URLs from index.html:
```bash
grep -oP 'href="https?://[^"]+' index.html | cut -d'"' -f2 | sort -u > .planning/phases/11-quality-assurance/urls.txt
```

2. Count total URLs to validate (expect 650+):
```bash
wc -l .planning/phases/11-quality-assurance/urls.txt
```

3. Validate URLs using parallel curl (20 connections, 10-second timeout):
```bash
cat .planning/phases/11-quality-assurance/urls.txt | \
    xargs -P 20 -I {} bash -c 'code=$(curl -s -o /dev/null -w "%{http_code}" --max-time 10 --user-agent "Mozilla/5.0 (compatible; QA-Bot/1.0)" "{}"); echo "$code {}"' | \
    tee .planning/phases/11-quality-assurance/url-validation-results.txt
```

4. Extract failures (4xx/5xx and timeouts):
```bash
grep -E '^[45][0-9]{2}|^000' .planning/phases/11-quality-assurance/url-validation-results.txt > .planning/phases/11-quality-assurance/broken-urls.txt || true
```

5. Review broken URLs list and categorize:
   - 404 (Not Found): need replacement
   - 403 (Forbidden): may work in browser (bot-blocked), verify manually
   - 429 (Rate Limited): retry after delay
   - 000 (Timeout): check URL manually, may be slow server

Note: If using a system without xargs -P, use GNU parallel instead:
```bash
cat urls.txt | parallel -j 20 'echo "$(curl -s -o /dev/null -w "%{http_code}" --max-time 10 "{}") {}"'
```
  </action>

  <done>All URLs validated with HTTP status codes logged; broken URLs identified in broken-urls.txt</done>
</task>

<task type="auto">
  <name>Task 2: Fix broken URLs in index.html</name>
  <files>index.html</files>
  <action>
For each URL in broken-urls.txt, find a working replacement and update index.html:

1. For each broken URL:
   a. Understand the context - what resource was it supposed to provide?
   b. Find working replacement from:
      - Original organization's current website
      - Wayback Machine (web.archive.org) if resource moved
      - Equivalent free resource that serves same learning purpose
   c. Verify replacement returns 2xx status code
   d. Update index.html inline using sed or direct edit

2. Replacement priority:
   - Same organization, updated URL (preferred)
   - Archive.org preserved version (if authoritative source important)
   - Equivalent alternative resource (if original org removed content)

3. For 403 errors (bot-blocked sites):
   - Verify URL works in browser
   - If it works for users, keep the URL (just bot-blocked)
   - If truly broken, find alternative

4. After all fixes, re-run validation on index.html to confirm no broken URLs remain:
```bash
grep -oP 'href="https?://[^"]+' index.html | cut -d'"' -f2 | sort -u | \

    grep -E '^[45][0-9]{2}|^000' || echo "All URLs valid"
```

Review resource URLs for paywall status and ensure free alternatives are available:

1. Identify common paywall domains in the URLs:
   - Known soft paywalls: medium.com, bloomberg.com, wsj.com, nytimes.com, ft.com
   - Known registration-walls: coursera.org (some courses), udemy.com
   - Research papers: journals that may have preprints available

2. Extract resources from learning path sections:
```bash
grep -B 2 -A 5 'Project Portfolio Ideas' index.html | \
    grep -oP 'href="https?://[^"]+' | cut -d'"' -f2 | sort -u > .planning/phases/11-quality-assurance/resource-urls.txt
```

3. Categorize URLs by paywall risk:
   - **Free confirmed**: .gov, .edu, open-source docs, official free APIs
   - **Audit needed**: medium.com, course platforms, news sites
   - **Likely free**: github.com, wikipedia.org, official documentation

4. For each audit-needed URL (sample 20-30 across levels):
   a. Open in incognito browser (avoid cookie-based access)
   b. Check for paywall indicators:
      - Login required to view content
      - "Subscribe to continue" messages
      - Limited free articles notice
   c. Document findings in paywall-audit.txt

5. For paywalled resources found:
   - Find free alternative that serves same learning purpose
   - Or add clarifying text: "(free tier available)" / "(registration required)"
   - Or substitute with equivalent free resource

6. Update index.html with:
   - Free alternative substitutions where needed
   - Clarifying annotations where appropriate

7. Create paywall audit log:
```

## Summary
- Resource URLs checked: [N]
- Paywalled content found: [N]
- Free alternatives substituted: [N]
- Annotations added: [N]

## Findings by Domain
[domain]: [free/paywalled] - [action taken]

## Substitutions Made
[original URL] → [replacement URL] - [reason]
```
